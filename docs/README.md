# Docs

This directory contains the foundational documentation for Hackathon Foundation.

Every document here defines *why* this project exists, *what* it aims to become, and *how* it operates. Before writing any code, creating any prompt, or designing any agent, these documents establish the principles that guide every decision.

## How to use these documents

1. Start with [VISION.md](./VISION.md) to understand the destination.
2. Read [MISSION.md](./MISSION.md) to understand the present focus.
3. Study [PHILOSOPHY.md](./PHILOSOPHY.md) to internalize the guiding principles.
4. Read [FREE_FOREVER.md](./FREE_FOREVER.md) to understand the non-negotiable constraints.
5. Study [ARCHITECTURE.md](./ARCHITECTURE.md) and [REPOSITORY_STRUCTURE.md](./REPOSITORY_STRUCTURE.md) to understand how the repository is built.
6. Read [COMPANY_MODEL.md](./COMPANY_MODEL.md) and [DEPARTMENTS.md](./DEPARTMENTS.md) to understand the organizational model.
7. Reference [RESPONSIBILITIES.md](./RESPONSIBILITIES.md) and [WORKFLOW_OVERVIEW.md](./WORKFLOW_OVERVIEW.md) when delegating tasks.
8. Reference [ENGINEERING_ROLES.md](./ENGINEERING_ROLES.md) for role-specific skills and example work.
9. Read [RULES.md](./RULES.md) and [SKILLS.md](./SKILLS.md) to understand how constraints and capabilities work together.
10. Use [TEMPLATES.md](./TEMPLATES.md) to select the right template for each deliverable.
11. Browse [AI_DIRECTORY.md](./AI_DIRECTORY.md), [FREE_MODELS.md](./FREE_MODELS.md), and [MODEL_SELECTION_GUIDE.md](./MODEL_SELECTION_GUIDE.md) to select the right model for each task.
12. Read [INTEGRATIONS.md](./INTEGRATIONS.md) to understand how to use the framework with your chosen AI tool.
13. Read [CONTRIBUTING.md](./CONTRIBUTING.md) and [COMMUNITY.md](./COMMUNITY.md) to understand how to participate.
14. Review the [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) before engaging with the community.
15. Refer to [FAQ.md](./FAQ.md) for quick answers to common questions.
16. Use [GLOSSARY.md](./GLOSSARY.md) to clarify unfamiliar terms.
17. Follow the [ROADMAP.md](./ROADMAP.md) to understand the project's current and future phases.
18. Reference [DESIGN_PRINCIPLES.md](./DESIGN_PRINCIPLES.md) and [CORE_CONCEPTS.md](./CORE_CONCEPTS.md) as needed during decision-making.

These documents are the constitution of the repository. Every phase, every file, every decision must be consistent with them.
