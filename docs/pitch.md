# Hackathon Foundation

## One-Problem. One-Solution. One-Standard.

### The Problem

AI coding assistants lose context every time you switch tools, models, or sessions. You explain the same project over and over. Progress is trapped in the tool that created it. Switching costs destroy productivity.

### The Solution

**Hackathon Foundation** is a universal, file-based workspace standard that makes AI context portable. Open any directory in any tool — CLI, IDE, or chat — and immediately see:

- What is being worked on.
- What was just done.
- What should happen next.
- What rules must be followed.

State files hold progress. Handoff protocols transfer context. Policies govern execution. Everything is in plain Markdown files — no databases, no servers, no lock-in.

### Why It Wins

| Challenge | Hackathon Foundation |
|---|---|
| Context lost on tool switch | State files survive every switch |
| Context lost on model switch | Export/import protocol preserves context |
| AI doesn't know project rules | Policies + handbook provide governance |
| Onboarding takes hours | State files explain everything in seconds |
| Fragmented workflow | Same structure works in every tool |

### The Impact

- **Before:** Each new AI session starts from zero. Context is rebuilt manually. Switching tools means losing progress.
- **After:** Any session starts from the last checkpoint. Tools are interchangeable. Context persists across the entire workflow.

### Free Forever

No paid models. No subscriptions. No proprietary formats. Open source. MIT licensed. Works with any AI tool on any platform.

### Built For

- Hackathons where teams switch between CLI, IDE, and AI chat tools.
- Open-source projects where contributors use different tools.
- Solo developers using multiple AI assistants.
- Teams that want to standardize how AI context is managed.
