# Contributing

See `docs/CONTRIBUTING.md` for the full contribution guide.

## Quick Reference

1. **Discuss.** Open an issue before starting work.
2. **Design.** Plan the contribution before writing.
3. **Implement.** Follow the repository standards.
4. **Review.** Self-review against the project's criteria.
5. **Submit.** Open a pull request with a clear description.
6. **Iterate.** Address review feedback.
7. **Document.** Contributions are recorded in `.memory/`.

## Standards

- All content must be free (no paid tools, models, or APIs).
- All content must be AI-agnostic (no tool-specific assumptions).
- All content must follow the naming conventions.
- All content must cross-link to related documents.
- All content must have a clear Purpose section.

See `docs/CONTRIBUTING.md` for detailed standards and workflow.
