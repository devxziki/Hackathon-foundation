# Engineering — Responsibilities

## Primary responsibilities

### Architecture and design

- Design system architecture that is modular, scalable, and maintainable.
- Make technology decisions within the constraints of the Free Forever policy.
- Document architecture decisions and trade-offs.
- Review and approve technical specifications.

### Implementation

- Build features according to specifications and design.
- Write code that follows the engineering principles in the handbook.
- Ensure code is self-documenting and follows project conventions.
- Handle edge cases and error states in all implementations.

### Quality

- Write unit tests and integration tests for all code.
- Participate in code reviews for all changes.
- Address technical debt proactively.
- Follow security and performance policies.

### Maintenance

- Fix bugs identified by QA, users, or monitoring.
- Refactor code that no longer meets quality standards.
- Update dependencies within policy guidelines.
- Maintain build and development tooling.

### Documentation

- Document APIs, modules, and architecture decisions.
- Keep inline documentation accurate and up to date.
- Provide technical input for user-facing documentation.

## What Engineering does NOT own

| Area | Owned by |
|---|---|
| UI/UX design | Design |
| User-facing documentation | Documentation |
| Quality policy | Quality Assurance |
| Deployment infrastructure | Operations |
| Technology research | Research |
