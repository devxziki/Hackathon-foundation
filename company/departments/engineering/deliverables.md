# Engineering — Deliverables

## Primary deliverables

### Feature implementation

Completed feature code that meets the specification, passes all tests, and has been reviewed. Includes:

- Source code following project conventions
- Unit and integration tests
- Updated API documentation
- Migration scripts or data transforms (if applicable)

### Architecture decisions

Recorded architecture decisions following the decision-making framework in the handbook. Each decision includes:

- Context and problem statement
- Options considered with trade-offs
- Decision and rationale
- Consequences and migration plan

### Technical specifications

Detailed specifications for features before implementation begins. Includes:

- System components and their interactions
- Data models and API contracts
- Performance and security considerations
- Integration points with existing systems

### Code reviews

Reviewed changes with actionable feedback. Each review covers:

- Correctness — does the code do what it should?
- Maintainability — is the code easy to understand and change?
- Compliance — does the code follow policies and conventions?
- Edge cases — are error conditions handled?

### Bug fixes

Fixed defects with root cause analysis. Each fix includes:

- Description of the bug and its impact
- Root cause
- Fix implementation
- Tests that prevent regression

## Deliverable quality gate

Every deliverable must pass these checks before leaving Engineering:

- [ ] Meets the definition of done
- [ ] Passes all automated tests
- [ ] Code reviewed and approved
- [ ] Documentation updated
- [ ] No regression in affected areas
