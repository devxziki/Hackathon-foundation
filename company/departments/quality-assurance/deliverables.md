# Quality Assurance — Deliverables

## Primary deliverables

### Test plans

Comprehensive plans for testing features and releases. Includes:

- Scope of testing (what is and is not tested)
- Test scenarios and test cases
- Test environment and data requirements
- Pass/fail criteria for each test

### Test reports

Results of test execution. Includes:

- Summary of tests run, passed, failed, and skipped
- Defects found with severity and priority
- Coverage metrics (code, feature, scenario)
- Release recommendation (approve, conditional, reject)

### Defect reports

Documented issues found during testing. Each report includes:

- Description and steps to reproduce
- Expected vs actual behavior
- Severity and priority classification
- Environment and version information
- Screenshots or logs where applicable

### Quality metrics

Measurable quality indicators. Includes:

- Test pass rate and trend
- Defect density per feature or release
- Regression incident rate
- Time to fix by severity
- Coverage percentages

### Release quality reports

Pre-release quality assessment. Includes:

- Summary of all quality gates and their status
- Open defects by severity
- Risk assessment for known issues
- Go/no-go recommendation with rationale

## Deliverable quality gate

Every deliverable must pass these checks before leaving QA:

- [ ] Based on clear, verifiable evidence (not assumptions)
- [ ] Accurately reflects the current state of the product
- [ ] Actionable — recipients can use it to make decisions
- [ ] Traceable to specific tests, requirements, or defects
