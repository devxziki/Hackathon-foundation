# Quality Assurance — Responsibilities

## Primary responsibilities

### Testing strategy

- Define the overall testing strategy: what to test, at what level, and how.
- Determine the appropriate mix of unit, integration, and end-to-end tests.
- Define test coverage targets and measurement methodology.
- Establish testing priorities based on risk assessment.

### Quality gates

- Define quality gates that must be passed before release.
- Verify that each gate is enforced in the development workflow.
- Grant or deny release approval based on gate status.
- Track gate compliance across releases.

### Test execution

- Write and maintain test plans for features and releases.
- Execute manual exploratory testing for new features.
- Run regression tests to verify existing functionality.
- Perform edge case and boundary testing.

### Defect management

- Track all defects in a centralized system.
- Classify defects by severity and priority.
- Verify fixes and close defects when confirmed resolved.
- Report defect trends and patterns.

### Process improvement

- Analyze defect patterns to identify process weaknesses.
- Recommend improvements to prevent recurring defect types.
- Review and update testing processes based on lessons learned.

## What QA does NOT own

| Area | Owned by |
|---|---|
| Feature implementation | Engineering |
| Code quality (static analysis, linting) | Engineering |
| Visual design correctness | Design |
| Documentation accuracy | Documentation |
| Infrastructure monitoring | Operations |
