# Quality Assurance — Collaboration

## Cross-department workflow

### Engineering → Quality Assurance

**Handoff:** Engineering provides feature implementations, bug fixes, and test infrastructure.

**QA returns:** Test results, defect reports, quality metrics, and release approval or rejection.

**Check:** QA verifies that all tests pass and quality gates are met before approving release.

### Design → Quality Assurance

**Handoff:** Design provides visual specifications, interaction patterns, and accessibility requirements.

**QA returns:** Visual regression reports, usability findings, and accessibility audit results.

**Check:** QA verifies that implementations match design specifications and meet accessibility standards.

### Documentation → Quality Assurance

**Handoff:** Documentation provides release notes, known issues pages, and troubleshooting guides.

**QA returns:** Verification that documented issues match actual known issues and instructions are accurate.

**Check:** QA tests documentation instructions against the product.

### Operations → Quality Assurance

**Handoff:** Operations provides test environments, deployment pipelines, and monitoring tools.

**QA returns:** Environment-specific test results, deployment verification, and monitoring integration feedback.

**Check:** QA verifies that deployments to each environment produce a working product.

### Research → Quality Assurance

**Handoff:** Research provides testing tool evaluations, automation recommendations, and methodology research.

**QA returns:** Adopted tools integrated into testing workflow, with feedback on effectiveness.

**Check:** QA evaluates research recommendations against testing requirements before adoption.

## Escalation path

| Issue | Escalate to |
|---|---|
| Quality gate dispute | QA Lead |
| Cross-department conflict | Department leads meeting |
| Severity classification dispute | QA Lead |
| Release blocking issue | CEO |
