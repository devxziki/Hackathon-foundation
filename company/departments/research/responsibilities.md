# Research — Responsibilities

## Primary responsibilities

### Technology evaluation

- Evaluate new tools, frameworks, and libraries for potential adoption.
- Assess each option against the Free Forever constraint (cost, licensing, limitations).
- Compare alternatives using consistent criteria.
- Document evaluation methodology and results.

### Proof-of-concept implementation

- Build small-scale prototypes to validate technology choices.
- Test integration with existing systems and workflows.
- Document implementation experience and findings.
- Provide migration guidance for recommended technologies.

### Best practice research

- Research industry best practices for engineering, design, testing, and operations.
- Evaluate practices against the company's context and constraints.
- Document recommended practices with rationale.
- Update recommendations as practices evolve.

### Tool and framework analysis

- Analyze tools and frameworks used by other departments for improvement opportunities.
- Identify gaps in the current toolchain.
- Research alternatives and improvements.
- Present findings with clear adoption recommendations.

### Knowledge base

- Maintain a repository of research findings and evaluations.
- Document lessons learned from past technology decisions.
- Provide reference information for ongoing projects.
- Make research accessible to all departments.

## What Research does NOT own

| Area | Owned by |
|---|---|
| Production code | Engineering |
| Design system | Design |
| Testing execution | Quality Assurance |
| Infrastructure management | Operations |
| Documentation publication | Documentation |
