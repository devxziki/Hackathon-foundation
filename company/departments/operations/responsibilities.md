# Operations — Responsibilities

## Primary responsibilities

### CI/CD pipelines

- Design and maintain continuous integration and delivery pipelines.
- Ensure builds, tests, and deployments run automatically on each change.
- Provide fast feedback on build and test failures.
- Manage deployment strategies (rolling, blue-green, canary).

### Infrastructure

- Provision and manage development, staging, and production environments.
- Manage infrastructure as code configurations.
- Ensure environment consistency across all stages.
- Plan and execute capacity management.

### Monitoring and observability

- Set up logging, metrics, and alerting for all services.
- Create dashboards for system health and performance.
- Define and measure service level indicators (SLIs).
- Establish incident response and on-call procedures.

### Security

- Implement infrastructure security best practices.
- Manage access control and secrets management.
- Apply security patches and updates.
- Support security audits and compliance requirements.

### Developer tooling

- Maintain development environment setup and tooling.
- Manage package registries and dependency mirrors.
- Provide build and deployment tooling.
- Document operational procedures and runbooks.

### Change management

- Manage release processes and deployment approvals.
- Maintain change logs and deployment records.
- Coordinate rollback procedures when needed.
- Document incident post-mortems.

## What Operations does NOT own

| Area | Owned by |
|---|---|
| Application code quality | Engineering |
| Application performance (code level) | Engineering |
| Test quality | Quality Assurance |
| Product security policy | CEO |
| Technology research and evaluation | Research |
