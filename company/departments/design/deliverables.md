# Design — Deliverables

## Primary deliverables

### Design specifications

Complete specifications for Engineering implementation. Includes:

- Wireframes or mockups for all screens and states
- Interaction patterns and micro-interactions
- Responsive behavior at all breakpoints
- Edge cases (empty states, error states, loading states)

### Design system tokens

Reusable design primitives for consistent implementation. Includes:

- Color palette with usage guidelines
- Typography scale and font specifications
- Spacing and sizing grid
- Shadow, border, and elevation tokens

### Component documentation

Specifications for reusable UI components. Includes:

- Visual appearance (all states)
- Behavior and interactions
- Accessibility requirements
- Usage guidelines and examples

### Usability review reports

Evaluations of implemented features. Includes:

- Comparison of implementation against design specification
- Identified visual or interaction issues
- Accessibility compliance notes
- Recommendations for improvement

### Prototypes

Interactive prototypes for user testing or stakeholder review. Includes:

- Clickable flows for key user journeys
- Annotations explaining interactions
- State transitions and feedback

## Deliverable quality gate

Every deliverable must pass these checks before leaving Design:

- [ ] Aligned with design system
- [ ] Accessibility requirements documented
- [ ] Edge cases covered (empty, error, loading states)
- [ ] Responsive behavior defined
- [ ] Handoff specification is unambiguous for Engineering
