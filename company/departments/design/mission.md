# Design — Mission

## Core mission

Create intuitive, accessible, and consistent user experiences that enable users to accomplish their goals efficiently.

## Why Design exists

Users judge a product by how it feels to use. Design bridges the gap between what the product does and how users interact with it. Without Design, features exist but lack clarity, consistency, and usability.

## Mission statement

Design delivers clear, accessible, and visually coherent interfaces that make complex workflows feel simple.

## Key objectives

1. **Usability first** — Every interface should be intuitive. Users should not need instructions to complete core tasks.
2. **Visual consistency** — Maintain a unified design system that ensures consistency across all surfaces.
3. **Accessibility** — Meet accessibility standards so the product can be used by everyone.
4. **Design fidelity** — Deliver specifications that Engineering can implement without ambiguity.
5. **Iterative improvement** — Refine designs based on user feedback and usage data.

## Non-goals

- Design does not implement code. It produces specifications for Engineering.
- Design does not define product strategy. It designs for the strategy defined by the CEO.
- Design does not write user-facing documentation. It provides input to Documentation.
- Design does not define quality policy. It follows accessibility and usability standards.
