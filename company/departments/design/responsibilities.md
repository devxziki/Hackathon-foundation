# Design — Responsibilities

## Primary responsibilities

### User experience

- Design user flows and interaction patterns for all features.
- Create wireframes, mockups, and prototypes for new functionality.
- Conduct usability reviews of implemented features.
- Define information architecture and navigation.

### Visual design

- Design the visual appearance of all product surfaces.
- Create and maintain color palettes, typography, spacing, and iconography.
- Ensure visual consistency across all screens and states.
- Design responsive layouts for all supported screen sizes.

### Design system

- Maintain reusable design tokens (colors, spacing, typography, shadows).
- Define component specifications for the component library.
- Document interaction patterns and usage guidelines.
- Version the design system alongside the codebase.

### Accessibility

- Ensure all designs meet accessibility standards (WCAG AA minimum).
- Provide accessible color contrast, focus states, and screen reader support.
- Review Engineering implementations for accessibility compliance.
- Create accessible interaction patterns for all components.

### Collaboration

- Provide clear design specifications to Engineering.
- Review implemented features for design fidelity.
- Participate in product reviews and provide design feedback.
- Document design decisions and rationale.

## What Design does NOT own

| Area | Owned by |
|---|---|
| Feature implementation | Engineering |
| User-facing text | Documentation |
| Accessibility testing | Quality Assurance |
| Design tooling infrastructure | Operations |
| UX research methodology | Research |
