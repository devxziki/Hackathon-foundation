# Documentation — Collaboration

## Cross-department workflow

### Engineering → Documentation

**Handoff:** Engineering provides API contracts, architecture documents, feature specifications, and changelog entries.

**Documentation returns:** API reference documentation, technical guides, and updated user documentation reflecting new features.

**Check:** Engineering reviews all technical documentation for accuracy before publication.

### Design → Documentation

**Handoff:** Design provides UI text, interaction descriptions, accessibility notes, and design system terminology.

**Documentation returns:** User-facing documentation that aligns with design language and describes interfaces accurately.

**Check:** Design reviews documentation for correct use of interface terminology.

### Quality Assurance → Documentation

**Handoff:** QA provides known issues, workarounds, troubleshooting steps, and edge case documentation needs.

**Documentation returns:** Updated troubleshooting guides, known issues pages, and release notes.

**Check:** QA verifies that documented workarounds are accurate and complete.

### Operations → Documentation

**Handoff:** Operations provides deployment procedures, environment configuration, infrastructure architecture.

**Documentation returns:** Deployment guides, environment setup documentation, and operations runbooks.

**Check:** Operations verifies that deployment documentation produces working environments.

### Research → Documentation

**Handoff:** Research provides findings, technology evaluations, and recommendations.

**Documentation returns:** Published research summaries, technology guides, and migration documentation.

**Check:** Research reviews documentation for technical accuracy and completeness.

## Escalation path

| Issue | Escalate to |
|---|---|
| Unclear documentation requirements | Documentation Lead |
| Cross-department conflict | Department leads meeting |
| Documentation standard dispute | Documentation Lead |
| Accuracy or correctness issue | CEO |
