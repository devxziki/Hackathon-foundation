# Documentation — Responsibilities

## Primary responsibilities

### User documentation

- Write and maintain user guides, tutorials, and how-to articles.
- Create getting started guides and onboarding materials.
- Document feature usage with clear examples.
- Maintain FAQ and troubleshooting sections.

### Technical documentation

- Document API endpoints, request/response formats, and authentication.
- Write architecture overviews and system design documents.
- Maintain configuration reference and environment setup guides.
- Document migration guides and upgrade paths.

### Contribution documentation

- Maintain contribution guides for open-source contributors.
- Document development setup and workflow.
- Create coding standards and style guides reference.
- Maintain community guidelines and code of conduct.

### Content management

- Organize documentation directory structure.
- Manage cross-references between documents.
- Ensure consistent formatting and style across all documents.
- Version documentation to match product releases.

### Quality

- Review documentation for accuracy, clarity, and completeness.
- Test documentation instructions against the actual product.
- Update documentation when features change.
- Remove or archive documentation for deprecated features.

## What Documentation does NOT own

| Area | Owned by |
|---|---|
| Product feature implementation | Engineering |
| UI text and labels | Design |
| Quality documentation | Quality Assurance |
| Deployment documentation | Operations |
| Research findings documentation | Research |
