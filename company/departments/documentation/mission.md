# Documentation — Mission

## Core mission

Create clear, accurate, and accessible documentation that enables users to use the product and contributors to build it.

## Why Documentation exists

A product is only as good as its documentation. Users need guides to understand functionality. Contributors need references to build effectively. AI systems need structured context to operate. Documentation makes all of this possible.

## Mission statement

Documentation delivers accurate, well-structured, and easy-to-navigate content that serves human and AI readers equally.

## Key objectives

1. **Accuracy** — Every document reflects the current state of the product. Outdated documentation is worse than no documentation.
2. **Clarity** — Documentation is written in plain language. Users should find answers without reading unrelated content.
3. **Completeness** — Every feature, API, and configuration option is documented.
4. **Discoverability** — Documentation is organized so readers can find what they need without prior knowledge.
5. **AI-readiness** — Documentation is structured for AI systems to parse and use as context.

## Non-goals

- Documentation does not define product features. It documents them.
- Documentation does not design user interfaces. It describes them.
- Documentation does not set engineering standards. It documents them.
- Documentation does not create visual assets. It requests them from Design.
