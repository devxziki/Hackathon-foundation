# Policies

## Purpose

The policies directory contains company-wide rules, constraints, and quality gates that all output must satisfy. Policies are non-negotiable and apply across all roles and departments.

## Responsibility

This directory enforces consistency and quality by defining the constraints that every deliverable must meet.

## What belongs here

- Coding rules — per-technology constraints (React, TypeScript, Tailwind)
- Security policies — security requirements and practices
- Testing policies — testing requirements and standards
- Git policies — version control conventions
- Quality gates — minimum quality requirements for deliverables
- Performance policies — performance targets and constraints

## What does NOT belong here

- Shared knowledge (belongs in `company/handbook/`)
- Reusable capabilities (belongs in `company/assets/`)
- Checklists (belongs in `company/assets/`)

## Future roadmap

This directory will be populated with rule and policy files for all engineering domains.
