# Skill Index

## Purpose

This index catalogs all capabilities the Hackathon Foundation workspace supports. Use it to understand what the system can do and which skill area applies to your current task.

## Skill categories

### Development

Build and maintain software across the full stack.

| Skill | Description |
|---|---|
| Architecture design | Define system structure, component boundaries, data flow |
| Backend implementation | Build server logic, APIs, data layers |
| Frontend implementation | Build user interfaces, state management, routing |
| API design | Define contracts, request/response formats, auth |
| Database design | Schema, migrations, query optimization |
| Testing | Unit, integration, e2e tests |
| Debugging | Root cause analysis, fix implementation |
| Code review | Correctness, maintainability, policy compliance |
| Refactoring | Restructure code without changing behavior |
| Performance optimization | Identify and resolve bottlenecks |

### Design

Create and maintain user-facing interfaces and experiences.

| Skill | Description |
|---|---|
| UI design | Visual mockups, component styling, layout |
| UX research | User flows, journey maps, usability analysis |
| Design system | Tokens, component specifications, patterns |
| Prototyping | Interactive mockups, clickable flows |
| Accessibility | WCAG compliance, screen reader support |
| Visual assets | Icons, illustrations, image creation |

### Documentation

Produce and maintain all forms of written content.

| Skill | Description |
|---|---|
| User documentation | Guides, tutorials, how-to articles |
| API reference | Endpoint docs, request/response examples |
| Technical writing | Architecture docs, system explanations |
| Release notes | Change summaries per release |
| Contribution guides | Setup, workflow, standards for contributors |
| Content review | Accuracy, clarity, completeness checks |

### Research

Explore, evaluate, and recommend technologies and practices.

| Skill | Description |
|---|---|
| Technology evaluation | Systematic tool and framework assessment |
| AI model evaluation | Capability, cost, limitation analysis |
| Best practice research | Industry patterns and methodologies |
| Proof of concept | Small-scale validation of technology choices |
| Migration planning | Step-by-step transition between technologies |
| Landscape monitoring | Track new releases and deprecations |

### Debugging

Diagnose and resolve issues across the system.

| Skill | Description |
|---|---|
| Root cause analysis | Systematic issue investigation |
| Bug reproduction | Reliable steps to trigger the issue |
| Fix implementation | Correct, tested resolution |
| Regression prevention | Tests that prevent recurrence |
| Performance diagnostics | Identify bottlenecks and inefficiencies |

### Quality Assurance

Verify that deliverables meet defined standards.

| Skill | Description |
|---|---|
| Test planning | Scope, scenarios, pass/fail criteria |
| Test automation | Automated test suites |
| Exploratory testing | Manual investigation of edge cases |
| Defect tracking | Documentation and classification |
| Release validation | Quality gate verification |

### Operations

Manage infrastructure and delivery pipelines.

| Skill | Description |
|---|---|
| CI/CD | Build, test, deployment automation |
| Environment management | Dev, staging, production configuration |
| Monitoring | Health, performance, error tracking |
| Incident response | Detection, diagnosis, resolution |
| Security | Access control, secrets management, compliance |

## How to use this index

1. Identify your current task category from the list above.
2. Open the relevant asset in `company/assets/` for templates, skills, or workflows.
3. Apply the relevant policy from `company/policies/`.
4. Execute using the execution lifecycle defined in `company/policies/execution-policy.md`.
