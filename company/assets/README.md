# Assets

## Purpose

The assets directory contains reusable internal capabilities that any employee (AI coding assistant) can use. These are the tools, templates, and processes that enable consistent, high-quality output.

## Responsibility

This directory provides the building blocks for executing engineering tasks. Resources are designed to be reusable across roles, projects, and sessions.

## What belongs here

- Templates — reusable structures for common deliverables
- Skills — step-by-step guides for common engineering tasks
- Workflows — repeatable processes for common scenarios
- Checklists — verification lists for quality assurance
- Playbooks — strategic guides for hackathon participation
- Prompts — reusable prompt templates for common tasks

## What does NOT belong here

- Company policies (belongs in `company/policies/`)
- Department definitions (belongs in `company/departments/`)
- Shared context (belongs in `company/handbook/`)

## Future roadmap

This directory will be populated with templates, skills, workflows, checklists, playbooks, and prompts for all common engineering tasks.
