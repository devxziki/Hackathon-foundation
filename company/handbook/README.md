# Handbook

## Purpose

The company handbook contains shared knowledge that every employee (AI coding assistant) must read before producing any output. It ensures consistency across all roles, sessions, and models.

## Responsibility

The handbook is the canonical source of truth for how the company operates. Every AI assistant reads the relevant sections before starting work.

## What belongs here

- Shared context — project goals, tech stack, folder structure
- Engineering standards — coding conventions, documentation standards
- Company philosophy — mission, vision, values
- Communication standards — how the company communicates internally

## What does NOT belong here

- Role definitions (belongs in `company/departments/`)
- Policies and rules (belongs in `company/policies/`)
- Reusable skills and templates (belongs in `company/assets/`)
- Public documentation (belongs in `docs/`)

## Future roadmap

This directory will be populated with:

- `context/` — Shared project context files
- `standards/` — Engineering and documentation standards
- `philosophy/` — Company mission, vision, and values
- `communication.md` — Communication standards and conventions
